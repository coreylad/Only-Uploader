rtorrent / ruTorrent Client Quick-Reference

Add a block like this to the TORRENT_CLIENTS section of your config to enable rtorrent as a download and upload client. The download_dir key is used by the cross-seed feature to tell rtorrent where to save content downloaded from source trackers. 

"rtorrent_main": {
    "torrent_client": "rtorrent",
    # XML-RPC endpoint – ruTorrent sets this up automatically.
    # Typical values:
    #   http://user:pass@localhost/RPC2          (local ruTorrent)
    #   https://user:pass@host.tld/rutorrent/plugins/httprpc/action.php
    "rtorrent_url": "http://user:pass@localhost/RPC2",

    # Where downloaded content from source-tracker cross-seeding is saved.
    # Must be accessible to both Only-Uploader and rtorrent.
    "download_dir": "/data/downloads/cross-seed",

    # Optional: path to rtorrent's session .torrent files (enables hash reuse).
    # "torrent_storage_dir": "/home/rtorrent/.session",

    # Optional label applied to every torrent added via this tool.
    # "rtorrent_label": "only-uploader",

    # Remote path mapping when running in Docker / on a remote host.
    # "local_path":  "/LocalPath",
    # "remote_path": "/RemotePath",
},

To use this client as the default set "default_torrent_client": "rtorrent_main" in the DEFAULT section, or pass --client rtorrent_main on the command line / in the upload form.
